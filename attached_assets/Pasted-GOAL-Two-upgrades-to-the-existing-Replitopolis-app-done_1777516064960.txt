   Technology: neon purple (#A855F7)
   Business: gold (#FFB800)

PASS 2 — UPGRADE THE VR / 3D SIMULATION GRAPHICS
═══════════════════════════════════════════════════════════════════

The existing career simulations look like a prototype. Polish the 
graphics without breaking simulation logic, scoring, or progress 
tracking.

DO NOT rewrite simulation gameplay. DO NOT change the data they save. 
DO NOT change which simulations exist or how they're routed. ONLY 
upgrade the visual layer.

For EACH simulation found in Step 0-B/0-C:

1. RENDERER UPGRADE
   - If using basic Three.js MeshBasicMaterial, upgrade to 
     MeshStandardMaterial with proper PBR (metalness, roughness, 
     albedo).
   - Enable shadows (PCFSoftShadowMap).
   - Add proper lighting: 1 directional sun light + 1 ambient + 
     environment HDR (use drei's <Environment preset="city" /> or 
     equivalent).
   - Enable tone mapping (ACESFilmicToneMapping) and correct color 
     space (SRGBColorSpace).

2. POST-PROCESSING (subtle, not overdone)
   - Bloom on emissive materials (glowing screens, neon signs).
   - SSAO or contact shadows for depth.
   - Slight vignette.
   - Use @react-three/postprocessing for React stacks.

3. ASSETS
   - Replace primitive placeholder cubes/spheres with low-poly GLTF 
     models from the public domain (Kenney.nl, Poly Pizza, 
     Quaternius). For each simulation, source 3-5 relevant props:
       AI/Tech sim: monitor, keyboard, server rack, robot arm
       Healthcare sim: hospital bed, IV stand, monitor, stethoscope
       Trades sim: toolbox, ladder, drill, workbench
       Engineering sim: blueprints, gears, crane model
       etc.
   - Load via useGLTF (drei) and cache.
   - Compress with draco if files exceed 1MB.

4. ENVIRONMENTS
   - Each simulation gets a proper room/setting instead of empty 
     space:
       Indoor sims: floor + 4 walls + ceiling, with windows showing 
       the Replitopolis skyline outside.
       Outdoor sims: ground plane with grass texture, sky dome, 
       distant city silhouette.
   - Use baseColor + normal map textures (free from polyhaven.com or 
       ambientCG) for floors, walls, surfaces. Avoid flat solid 
       colors.

5. CHARACTERS IN SIMULATIONS
   - If sims use placeholder character meshes, swap to Mixamo-style 
     rigged low-poly characters or Ready Player Me avatars.
   - Match the high-school-age aesthetic from Pass 1.
   - Add idle and interaction animations from Mixamo (free).

6. UI INSIDE SIMULATIONS
   - Sim HUD must use the same style as the rest of the app: dark 
     navy panels, neon accent colors per district, glowing text, 
     same font.
   - Add a "Pause" button, "Exit Simulation" button (returns to 
     city), and a progress bar.

7. PERFORMANCE GUARDRAILS
   - Cap simulation scenes at 60fps target.
   - Use LOD for any complex GLTF.
   - Lazy-load each simulation route — never load all sim assets at 
     once.

8. DO NOT BREAK
   - Existing simulation scoring logic
   - Progress saving / Career Passport updates
   - Any database writes the simulations make
   - Existing simulation routes
